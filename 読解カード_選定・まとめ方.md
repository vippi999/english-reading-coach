# 読解カード｜選定・まとめ方

このファイルだけを、読解カードの選定・文面・具体例・JSON形式の正本とします。`手順_実読チャット.md`、`指示_実読チャット.md`、過去の実読報告には同じ基準を書きません。内容が違う場合は、このファイルを優先します。

## 1. 何をカードにするか

カードは、次の長文で使うための短い私的メモです。単語、熟語、チャンク、ディスコースマーカー、構文、数量表現などを対象にします。

### ユーザーがカード内容を確定した場合

- その言い方と短さをそのまま使う
- AIが一般的・教科書的な表現へ直さない
- 確実な誤りがあれば指摘するが、ユーザーの確認なしに修正しない
- 一度確定したカードを、AI判断で再要約しない

### ユーザーが確定していない場合

AIは次のどれかに当てはまり、今後も覚えた方がよいと判断したものを、この手順に従ってカードにしてよいです。

- 知らなかったため文が崩れた
- 知っていたが、速度内で意味や働きが出なかった
- 語義・品詞・切れ目を誤った
- 文と文の関係を決める合図だった
- 別の長文でも使う可能性がある

頻出度だけで落としません。一方、次はカードにしません。

- 正しく即時処理でき、復習も不要なもの
- AIが念のため足した一般知識
- その長文だけの事実
- 白紙化、早期確定、保持失敗などの症状そのもの

症状は実読報告へ残します。症状を起こした具体的な語句・構文に再利用価値がある場合だけ、その知識をカードにします。

## 2. カードの書き方

- 1枚につき、思い出す中心は1つ
- 本人の言葉、比喩、覚えやすさを優先する
- 短い標語へ削りすぎず、本人に通じる自然な一文にする
- 付随説明、例外、文法用語を足しすぎない
- 単純語義だけなら `largely：ほとんど` のように、コロンの後ろを隠す
- 文法なら `冠詞・形容詞・名詞の順番はよくある。` のような短い一文にする
- ディスコースマーカーは日本語訳の暗記ではなく、`After allは前の内容の理由を出す` のように働きを残す

7月13日に確定した書き方の例:

- `did + 原形は、実際にそうだったんだと強調している。`
- `文頭のStillは、「それでも」という逆説を表す。`
- `After allは、前の内容の理由を出すときに使う。`
- `fourthは4つではなく4番目なので、一つの順位を意識する。`
- `outsideは前置詞にもなれて、inは内、outsideは外という対比になる。`
- `due toは右向き矢印。`

## 3. 具体例

各カードに、最低1件の `短い英語＋その部分の日本語` を付けます。

- 完全な文でなくてよい
- 元の英文断片だけで思い出せるなら、その断片を優先する
- 日本語は、その英語部分に対応させる
- 読み方メモや付随説明を追加しない

例:

```text
英語: both in and outside the United States
日本語: アメリカ国内と国外の両方で
```

## 4. 保存形式

正本はプロジェクト直下の `review_items.json` です。実践チャットは `app/public/review_items.json` を編集しません。

各カードは次の形を使います。

```json
{
  "id": "rules1-l2-largely",
  "source": {
    "book": "The Rules 1",
    "unit": "Lesson 2",
    "location": "位置未確認",
    "excerpt": "largely positive"
  },
  "target": "largely",
  "category": "word_or_chunk",
  "prompt": "largely の意味を確認する。",
  "answer_core": "ほとんど",
  "usage_examples": [
    {
      "english": "largely positive",
      "japanese": "ほとんど肯定的な"
    }
  ],
  "card_sentence": {
    "before": "largely：",
    "hidden": "ほとんど",
    "after": ""
  },
  "initial_processing": "意味が出なかった",
  "cue": "",
  "created_at": "2026-07-13",
  "learned_on": "2026-07-13"
}
```

補足:

- `id`は英小文字・数字・ハイフンで重複させない
- `category`は `discourse_marker` / `word_or_chunk` / `grammar_or_quantity`
- `card_sentence.hidden`だけをタップまで隠す
- `answer_core`、`initial_processing`、`cue`は必要最小限にする
- `usage_examples`は英語と日本語だけにする
- 既存の追加フィールドは削除せず保持する

## 5. 実践チャットの処理順

1. このファイルとGitHub上の最新 `review_items.json` を読む
2. 同じ知識のカードがないか確認する
3. ユーザーが確定した内容なら、そのままカードにする
4. 未確定の候補なら「1. 何をカードにするか」で選ぶ
5. 「2. カードの書き方」と「3. 具体例」に従って作る
6. `review_items.json`へ追加または更新する
7. 実読報告へカードIDを記録し、「保存済み」とだけ伝えて実読を続ける

カード確定のたびに保存し、複数Lessonの終了や司令塔の返答を待ちません。アプリへの配備は司令塔が節目でまとめて行います。

## 6. この基準の変更

ユーザーがカードの選び方・短さ・表現について直接指示した時だけ、このファイルを更新します。過去の報告MDは観測記録であり、現行ルールにはしません。
