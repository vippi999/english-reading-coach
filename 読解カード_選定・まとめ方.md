# 読解カード｜選定・まとめ方

作成: 2026-07-13 / 対象: 実践チャット、司令塔、将来の読解デッキ

## 1. 読解カードとは

英語長文を実際に読んだ時、意味が出なかった、誤った、または速度内で使えなかった知識を、次の長文で使える形にした短い私的メモです。

単語だけでなく、次を含みます。

- 単語の文中語義・品詞
- 熟語、チャンク、語の切れ目
- ディスコースマーカーと前後関係
- 基本動詞構文、時制、数量表現
- 句読法など、別の長文でも再利用できる読み方

1件を「読解カード」、全体を「読解デッキ」と仮称します。

## 2. 最優先の考え方

段落を見返した時、ユーザーが「ここを知らなかったため文が崩れた」と分かる箇所を出発点にします。AIが全文からカード候補を大量に探すことを主作業にしません。

カードは一般向け教材ではなく、本人だけが使うものです。カード表面は問題文と解答欄に分けず、本人の言葉に近い、少しゆるい自然な一文として残します。短い標語や普通の単語帳へ圧縮しすぎず、一読して意味の流れが分かることを優先します。正確な説明、根拠、例外は詳細欄または実読報告へ残します。

## 3. カードにする基準

次のいずれかに当てはまれば候補にします。

- 知らなかったため文意が崩れた
- 知っていたが、速度内で意味や機能が出なかった
- 別の語義・品詞・切れ目へ誤確定した
- 文と文の関係を決める重要な合図だった
- 別の教材でも再利用する可能性がある
- ユーザー本人が、今後も見返したいと判断した

1回しか出ない表現でも、今回の読解を明確に崩した、または本人が残したい場合は登録して構いません。頻出度だけで落としません。

## 4. カードにしないもの

- 長文の内容にしか関係しない固有の事実
- 正しく即時処理でき、本人も復習不要と判断したもの
- AIが念のため追加した一般知識
- 白紙化、早期確定、保持失敗などの症状そのもの
- 正確な説明がまだ確定していない項目

白紙化や早期確定は重要ですが、知識カードではなく実読報告へ記録します。その現象を起こした具体的な単語・構文に再利用価値があれば、その知識だけをカードにします。

## 5. 1カード1中心

1枚で思い出す中心は原則1つです。

- `After all` の中心: **理由を示す**
- `Still` の中心: **逆接・反転**
- `did + 原形` の中心: **強調**

カード表面に複数の説明を並べません。関連情報が中心の理解に不可欠なら詳細へ置きます。別々に思い出す必要がある場合だけカードを分けます。

## 6. 何を見せ、何を隠すか

### 常に見せる

- 本人の言葉に近い自然な一文。ただし、今回最も思い出したい部分だけは空欄にする
- 種類
- 出典

### タップするまで隠す

- 一文の中にある、今回最も思い出したい部分

### 詳細を開いた時だけ見せる

- 正確な意味・構造
- 最初の誤り
- 見分ける条件
- 元の英文断片

隠すのはカード全体や回答欄ではなく、一文の中の中心部分だけです。空欄には下線を出し、その場所をタップすると同じ文字サイズで内容が現れる形にします。

## 7. 種類別のまとめ方

### A. ディスコースマーカー

日本語訳を一語で答えさせず、前後関係での機能を核にします。

```text
表示: After all
隠す: 理由を示す
詳細: 直前の判断・主張の理由を後ろから補う
```

```text
表示: Still
隠す: 逆接・反転
詳細: 前の内容を認めたうえで「それでも」と切り替える
```

`After all` は結果ではなく、今回の中心は理由提示です。

### B. 単語

単語単体の全語義ではなく、今回の文で必要だった語義・品詞を核にします。

```text
表示: prompt A
隠す: Aを促す／引き起こす
詳細: 今回は動詞。最初は「即座の」を当てた
```

### C. 熟語・チャンク・切れ目

どこまでが1まとまりかと、その文中意味を短くします。

```text
表示: throughout their lives
隠す: 生涯を通じて
詳細: their livesが名詞句。there livesと切らない
```

### D. 構文・文法

名称の暗記より、長文中で何を表す形かを核にします。

```text
表示: did + 原形
隠す: 実際に〜した（強調）
詳細: 肯定文でdo/did + 原形なら強調を疑う
```

### E. 前置詞・位置関係

本人に通じる短い対比を使って構いません。

```text
表示: outside
隠す: 前置詞にもなる
詳細: in=内 / outside=外。in and outside the United States
```

### F. 数量・比較

式や倍率を短く示し、必要なら比較元保持の注意を詳細へ置きます。

```text
表示: thirty times greater
隠す: 30倍
詳細: 比較元の数値も保持して検算する
```

## 8. 一文へまとめる基準

カード全体は、本人が口頭で言った説明を優先します。覚えやすさにつながる本人独自の表現や比喩を、一般的・教科書的な説明へ置き換えません。確実な誤りを見つけた場合は指摘しますが、本人の確認なしにカード文を直しません。

良い例:

- `After allは、前の内容の理由を出すときに使う。`
- `文頭のStillは、「それでも」という逆説を表す。`
- `outsideは前置詞にもなれて、inは内、outsideは外という対比になる。`
- `fourthは4つではなく4番目なので、一つの順位を意識する。`

避ける例:

- 辞書の語義をすべて列挙する
- 例外・文法用語・注意事項を表面へ詰め込む
- 日本語訳を一語に固定すると機能が消えるのに、訳だけを正答にする
- 1枚で複数の無関係な知識を覚えさせる
- `After all → 理由`のように、本人の説明から文を取り除いて標語へ戻す

単純に単語の意味を覚えていないだけの項目は例外的に `語: 意味` とし、コロンの後ろを隠します。

## 9. 作成手順

1. ユーザーが段落を見返し、分からなかった箇所を申告する
2. 初読時にどう処理したかを記録する
3. 教材解説や参考書で、正しい意味・機能を確定する
4. 再利用できる知識か、本人が残したいかを確認する
5. 最も思い出したい1点を、短い私的メモにする
6. 正確な説明と最初の誤りを詳細へ分ける
7. 実際の使われ方が分かる具体例と、その例全体の日本語訳を作る
8. `review_items.json` の最新版を読み、重複を確認して追加・更新する
9. 実読報告へ登録したカードIDを残す

## 10. 現在のアプリへ渡すJSON形式

教材横断カードの正本は、プロジェクト直下の `review_items.json` です。実践チャットはこのファイルだけを更新し、`app/public/review_items.json` は編集しません。アプリ起動・ビルド時に正本から自動コピーします。

アプリが各カードで必須とする項目は次です。

- `id`: 英小文字・数字・ハイフンによる重複しないID。例 `rules1-l2-largely`
- `source.book`: 教材名
- `source.unit`: Lesson名
- `source.location`: 段落・行番号。分からないときは `位置未確認` と明記する
- `source.excerpt`: 必要最小限の英文断片
- `target`: 対象表現
- `category`: `discourse_marker` / `word_or_chunk` / `grammar_or_quantity`
- `prompt`: 詳細画面で使う確認質問
- `answer_core`: 明白な誤りを避けた正確な説明
- `usage_examples`: 具体例の配列。最低1件を必須とする
  - `english`: 対象表現の使われ方が分かる英語。完全な文でも、意味のある句・節でもよい
  - `japanese`: `english` 全体に対応する自然な日本語訳
- `card_sentence.before`: 隠す部分より前
- `card_sentence.hidden`: タップまで隠す、本人が最も思い出したい部分
- `card_sentence.after`: 隠す部分より後ろ
- `initial_processing`: 初読時に実際に行った誤処理。誤処理がなければ、その知識が出なかった事実を書く
- `cue`: 次に見分けるための条件。不要なら空文字列
- `created_at`: JSON項目を作った日
- `learned_on`: 実際にそのLessonを読み、知識を確定した日。省略時は `created_at` を使う

`prompt_mode`、`accepted_keywords`、`status`、`review_history` は正本に残して構いません。アプリ表示は必須項目を使い、未使用の追加項目は壊さず保持します。

具体例は短い英語と、その部分に対応する日本語だけにします。完全な文である必要はなく、元の英文断片だけで思い出せるなら、その断片を優先します。例だけで足りる情報を説明文として重ねません。本人の言い方・短さ・覚えやすさを正確さより優先し、確実な誤りがある場合だけ指摘して、勝手には直しません。

単純語義カードの例です。ユーザーの表現を優先し、コロンの後ろだけを隠します。

```json
{
  "id": "rules1-l2-largely",
  "source": {
    "book": "The Rules 1",
    "unit": "Lesson 2",
    "location": "位置未確認",
    "excerpt": "largely"
  },
  "target": "largely",
  "category": "word_or_chunk",
  "prompt": "largely の意味を確認する。",
  "answer_core": "ほとんど／主に。今回ユーザーが残す表現は『ほとんど』。",
  "usage_examples": [
    {
      "english": "The area is largely rural.",
      "japanese": "その地域は大部分が農村地帯だ。"
    }
  ],
  "card_sentence": {
    "before": "largely：",
    "hidden": "ほとんど",
    "after": ""
  },
  "initial_processing": "意味が出なかった",
  "cue": "程度や割合を大きくまとめる副詞として確認する",
  "created_at": "2026-07-13",
  "learned_on": "2026-07-13"
}
```

文法カードの例です。一般化した一文のうち、思い出したい中心だけを隠します。

```json
{
  "id": "rules1-l2-article-adjective-noun-order",
  "source": {
    "book": "The Rules 1",
    "unit": "Lesson 2",
    "location": "位置未確認",
    "excerpt": "該当する英文断片"
  },
  "target": "冠詞＋形容詞＋名詞",
  "category": "grammar_or_quantity",
  "prompt": "名詞句のよくある語順を確認する。",
  "answer_core": "冠詞・形容詞・名詞の順番はよくある。",
  "usage_examples": [
    {
      "english": "She bought a small red car.",
      "japanese": "彼女は小さな赤い車を買った。"
    }
  ],
  "card_sentence": {
    "before": "",
    "hidden": "冠詞・形容詞・名詞",
    "after": "の順番はよくある。"
  },
  "initial_processing": "初読時の実際の処理を書く",
  "cue": "一問だけの説明にせず、別の英文でも使える名詞句の順番として見る",
  "created_at": "2026-07-13",
  "learned_on": "2026-07-13"
}
```

問題文固有の答えへ寄せすぎず、別の長文でも再利用できる範囲で一般化します。ただし、ユーザー本人の言い方・比喩・覚え方を最優先で残します。明白な意味違い、品詞違い、事実誤認を見つけた場合は指摘しますが、本人の確認なしにカード文を修正しません。カードの一文を教科書的な長文へ変えません。

同じ知識が別教材で再登場したときの複数出典表示は未実装です。現時点では重複カードを勝手に作らず、実読報告へ再登場を記録して司令塔へ渡します。

## 11. 実践チャットの責任範囲

実践チャットは、ユーザーが申告した項目をもとに、本人の言い方を残した短いカード案と、短い具体例・日本語訳を作り、GitHubへ保存します。カードの価値を頻出度だけで勝手に否定しません。確実な誤りがあれば指摘しますが、本人の表現を勝手に教科書的な表現へ直しません。

一方で、早期確定・白紙化・保持失敗などの長期的な原因診断や、学習計画の変更は司令塔へ渡します。

ユーザーが実践チャットでカードの選定・短縮方法について直接指示した場合は、そのチャットからこのMDを更新できます。AIの独自判断だけで基準を変更しません。
