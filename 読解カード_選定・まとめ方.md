# 読解カード｜選定・まとめ方

このファイルだけを、カードの選定・文面・具体例・JSON形式の正本とします。他の指示や手順へカード方針を重複して書きません。

## 1. 最優先

カードは教科書や辞書ではなく、ユーザー本人が長文中で思い出すための私的な復習道具です。正確さや網羅性より、本人が一目で理解し、記憶を呼び戻せることを優先します。

- 本人が確定した言い方と短さをそのまま使う。
- 意味や働きを意図的に広く、短く、直感的にまとめてよい。
- `as`や`out of`を広くまとめても、「一般化しすぎ」と評価しない。
- `vital`のように、結びつけて覚えやすい複数の意味を1枚へ入れてよい。
- 明確な誤りだけ指摘し、本人の確認なしに教科書的な表現へ直さない。
- カード表面では、`A`・`B`・`S`・`V`・`S′V′`・`〜`などの抽象記号をなるべく使わない。
- 自然な日本語、実際の短い英語、具体例、本人に通じる矢印や比喩を優先する。
- 条件、例外、付随説明を勝手に足さない。詳しい確認は参考書へ戻り、カードは思い出しやすく保つ。

## 2. 選定

次の長文でも使う可能性があり、次のいずれかに当てはまるものをカードにします。

- 知らなかったため文が崩れた。
- 知っていたが、速度内で意味や働きが出なかった。
- 語義、品詞、切れ目を誤った。
- 文と文の関係を決める合図だった。
- ユーザーが復習したいと判断した。

次はカードにしません。

- 正しく即時処理でき、本人も復習不要と判断したもの。
- AIが念のため足した一般知識。
- その長文だけの事実。
- 白紙化、早期確定、保持失敗などの症状そのもの。

症状を起こした具体的な語句や構文に再利用価値がある場合は、その知識だけをカードにします。

## 3. 文面と具体例

- 思い出す中心が本人に分かる形にする。本人が複数語義をまとめたい場合は分割しない。
- 単純語義は `largely：ほとんど` のように書く。
- 文法や働きは、本人に通じる自然な一文にする。
- ディスコースマーカーは、必要なら日本語訳より前後での働きを残す。
- 各カードに最低1件、短い英語と対応する日本語を付ける。
- 完全文でなくてよい。元の英文断片で思い出せるなら優先する。
- 具体例へ読み方メモや追加説明を混ぜない。

本人が確定した例:

- `did + 原形は、実際にそうだったんだと強調している。`
- `After allは、前の内容の理由を出すときに使う。`
- `outsideは前置詞にもなれて、inは内、outsideは外という対比になる。`
- `due toは右向き矢印。`

## 4. 保存形式

正本はプロジェクト直下の `review_items.json` です。`app/public/review_items.json` は編集しません。

```json
{
  "id": "rules1-l2-largely",
  "source": {
    "book": "The Rules 1",
    "unit": "Lesson 2",
    "location": "位置未確認",
    "excerpt": "largely positive"
  },
  "target": "largely",
  "category": "word_or_chunk",
  "prompt": "largely の意味を確認する。",
  "answer_core": "ほとんど",
  "usage_examples": [
    {
      "english": "largely positive",
      "japanese": "ほとんど肯定的な"
    }
  ],
  "card_sentence": {
    "before": "largely：",
    "hidden": "ほとんど",
    "after": ""
  },
  "initial_processing": "意味が出なかった",
  "cue": "",
  "created_at": "2026-07-13",
  "learned_on": "2026-07-13"
}
```

- `id`は英小文字・数字・ハイフンで重複させない。
- `category`は `discourse_marker`、`word_or_chunk`、`grammar_or_quantity` のいずれか。
- `card_sentence.hidden`だけをタップまで隠す。
- `answer_core`、`initial_processing`、`cue`は必要最小限にする。
- `usage_examples`は英語と日本語だけにする。
- 既存の追加フィールドは削除しない。

## 5. 保存手順

1. 最新の `review_items.json` を読む。
2. 同じ知識の既存カードがないか確認する。
3. 本人が確定した表現を優先して追加または更新する。
4. 同じ段落で確定したカードをまとめて一度で保存する。
5. 保存後は「保存済み」と伝えて実読へ戻る。

この基準は、ユーザーがカード方針を直接変更した時だけ更新します。過去の実読報告は現行ルールにしません。
