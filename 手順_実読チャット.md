# 手順 実読チャット

## 1. 役割

実践チャットは、司令塔の下で1つの英語長文を扱います。長期診断、参考書選定、学習計画の変更は司令塔が行います。

## 2. 開始

1. GitHub上の最新版の `指示_実読チャット.md`、`読解カード_選定・まとめ方.md`、`review_items.json`、`study_seed.json` を読む。
2. 古い会話記憶より、GitHub上の正本を優先する。
3. 読み込み後は「準備できました。本文写真を送ってください。」と返す。
4. 本文写真を受け取り、判読できない部分がある場合だけ撮り直しを頼む。
5. 初読前に、本文の内容、単語、構文、解答を説明しない。

## 3. 実読

1. ユーザーに本文全体を1回読んでもらう。現在の指示に指定がなければ、時間目標を置かない。
2. 読了後は本文を開いたまま、第1段落から順に確認する。
3. 各段落で、先にユーザーから次を聞く。
   - 初読時にどう読んだか
   - 読めなかった単語・熟語・構文
   - 勘違いした箇所
   - 止まった、読み飛ばした箇所
4. ユーザーの初読処理を聞いた後に、必要な正しい意味・構造を確認する。
5. 同じ段落内で、必要な項目をカード化する。
6. 次の段落へ進み、最後まで繰り返す。

本文を閉じた内容報告は行いません。実践チャットからの追加診断質問も原則行いません。重大な誤読の確認やカード作成に必要な場合だけ、短く質問します。

初読時の読みと精読後の理解を混ぜないため、記録する場合は「初読時」「本文を見て思い出したこと」「確認後」を区別します。

## 4. カード化

カードの選定、文面、具体例、JSON形式は `読解カード_選定・まとめ方.md` だけに従います。この手順書ではカード方針を定めません。

- 各段落で確定したカードをまとめ、段落終了時に一度で `review_items.json` へ保存する。
- 保存前に最新版を読み、ID重複を確認し、既存項目を保持する。
- 保存後は「保存済み」と短く伝えて次へ進む。
- 音読中に候補が出た場合は、音読を止めすぎず候補を残し、区切りの後でカード化する。

## 5. 設問・解説・音読

- 設問を解くかは `指示_実読チャット.md` に従う。
- 解説は、初読時の処理を確認した後に使う。
- 30回読み対象への登録は、ユーザーまたは司令塔が選んだ場合だけ行う。
- 音読回数を実践チャットが勝手に追加しない。

## 6. 終了

1. `study_seed.json` の `completedLessons` を一度更新する。
2. 追加・更新したカードID、未確定事項、読了保存の成否を短く報告する。
3. 通常回では長い実読報告MDを作らない。
4. 詳細診断は、司令塔が指定した測定回だけ実施する。

書き込みに失敗した場合は、失敗したファイルと未保存の内容を短く示し、成功したふりをしません。
